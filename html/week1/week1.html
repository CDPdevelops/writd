<Summary/>

Development Enviroment
The first step was to become accustomed to a development enviroment / text editor. I became familiar with Sublime Text, a shareware cross-platform source code editor with a Python application programming interface. I liked the way each element of html, CSS and Javascript was colour coded and easily distinguishable, with the ease of finding errors.
Figure 1
File Structure
Secondly I began setting up the file structure stored in the practical_0 folder. A CSS folder to store all external stylesheets removing any inline styling, js folder to store any javascript files, media folder to store any media files such as images or videos and the HTML file index.html that will be launched initially into any web browser.
Figure 2
Implementing a webpage
I implemented the mockup webpagef using the relevant resources from lecture notes and w3schools.com. With my display I was able to split screens keeping index.html and the source code on Sublime Text opened at the same time. As I updated and saved my code I would refresh the webpage and see any changes made.
This mockup webpage is used to gather information acting as book store checkout service, recording what books customers purchase. The page consists of a header and an input form held within the body (styled by the CSS styling sheet). Basic CSS styling stored in the style.css file helped me obtain a mirror image representation with the example shown in the Practical PDF, removing any inline styling.
Figure 3
Following w3schools and lecture notes, the correct html format was used including html declaration, html element, head and body in the index.html file. I started by giving the webpage a title that would be visible in the browsing window 'practical_0' as shown in practical solution. Next I added any meta information located in the head, UTF-8 (telling the browser to use the UTF-8 character encoding), the responsive meta tag and linking the stylesheet style.css.
Next I made a header element that contained the title and JQuery Logo followed by a hr element seperating the header from the form. The logo's size properties was edited in the style.css likewise the body's padding (generating space around the content).
The input form consisted of customer details, payment & contact method, further comments section and shopping kart displaying items they are about to purchase. Customer details used placeholders with different input types: text, email and postcode. The shopping kart table used th, td and tr elements, the pound sign used the entity name '&ound' and a border value 1 to give the table an outline. The payment method used radio buttons as inputs and the contact section used checkboxes. The final comment section was just a large text area placeholder that was ID'd as 'textbox' and referenced in the style.css.
Finally the last step was to add a reset and submit button to clear and submit the form. As the form action is blank this form will not be sent to any server upon clicking the submit button but the reset button will clear any placeholders containing any data.
 

<Further Information/>

Placeholders and their maxLength/patterns
I became more familiar with input forms alongside w3schools help, I understood how input forms may be validated when entering data into the placeholders. The placeholders can utilise the required atribute meaning that data must be entered and if left blank, a notifaction above the placeholder will tell the user they must "fill out this field" or the form can not be submitted. The maxLength can also be distinguished, meaning only a set amount of characters can be entered, also the customer name, street address, email address, postcode and phone number use a pattern atribute. W3schools state: "The pattern attribute specifies a regular expression that the input element's value is checked against on form submission"(W3 Schools). The placeholder also can contain a short hint before the user enters a value (e.g. UK Postcode required, XX11 1XX).
  Figure 4
Placeholder maxLength Pattern Meaning
Customer Name 70 [A-Za-z]{1,70} Letters, no more than 70 characters as it's firstname and surname.
Street Address 50 [A-Za-z0-9]{1,50} Letters and numbers, no more than 50 characters.
Email Address 62 [a-z0-9._%+-]+@[a-z0-9.-]+\.[a-z]{2,}$ Email address format
UK Postcode 8 [A-Za-z]{1,2}[0-9Rr][0-9A-Za-z]? [0-9][ABD-HJLNP-UW-Zabd-hjlnp-uw-z]{2} UK Postcode Format
UK Number 11 ^\s*\(?(020[7,8]{1}\)?[ ]?[1-9]{1}[0-9{2}[ ]?[0-9]{4})|(0[1-8]{1}[0-9]{3}\)?[ ]?[1-9]{1}[0-9]{2}[ ]?[0-9]{3})\s*$ UK mobile/landline format
