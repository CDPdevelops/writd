Analyze and Explain
Explain the following JavaScript statements and what would be the result of each statement. When JavaScript uses different operators they can be distinguished into different categories. Assignment, comparison, arithmetic, string, and logical types
Figure 1
JavaScript Statement Operators Type Example Result of myAge
var myAge =45; "=" Assignment x = y The var statement declares a variable. In this case the myAge variable has been set an initial value of 45.
myAge += 4; "+=" Assignment x = x + y With the "+=" operator, the myAge value alters by adding 4 to the initial value. The new myAge value is 49.
myAge ++; "++" Arithmetic x = ++y With the "++" incremental operator, the myAge value alters by +1 meaning the myAge value is 46.
myAge == 40+10; "==" Comparison x == y With the "==" operator, the myAge value is comparing x == y. The logical result returned is false, given that myAge is 45 and not equal to 50.
myAge <=38; "<=" Comparison x <= y With the "<=" operator, the myAge value is comparing x is less than or equal to y. The logical result returned is false, given that myAge is 45 and is more than 38.
myAge =40; "=" Assignment x = y With the "=" operator, the myAge value alters to 40 as it has been assigned a new value.
 
Hands on with JS
The next task was to create an HTML document with JavaScript code.
Figure 2
The var statement declares a variable. In this case the myAge variable has been set an initial value of 45 using the assignment operator. Next the myAge value changes positively by 4 using another assignment operator, meaning the new value of myAge is = 49. The myAge has been assigned a new value containing a string value and a myAge numeric value. This can be outputted on screen by using the following code: myAge = "My age is" + " " + myAge; The blank quotation marks allow a space between the string and numeric value. The write() method writes HTML or JavaScript code to a document. Outputted message on screen: My age is 49.
The next step was adding the rest of the statements in Task 1 one by one after 'myAge +=4' and check the results against the initial explanation, showing any discrepancies. All assignment and arithmetic statements showed no discrepancies and the correct value was shown. For the comparison statements when asked to compare they both outputted the same value instead of logically stating wether the statement was true or false. They stated the initial value that had been assigned: My age is 45.
Figure 3
I had changed the code in order to obtain the logical answer, true or false. Instead of locating the 'myAge <= 38' after the var myAge variable, I moved the statement into a set of brackets after the statement: "document.getElementById("age").innerHTML =", furthermore adding a paragraph tag located outside before the script using the id tag "age". This allowed a logical value to be shown, the myAge value is comparing x is less than or equal to y. The logical result returned is false, given that myAge is 45 and is more than 38. The same applied for the statement 'myAge == 40+10'. This is comparing whether or not myAge is equal to 40+10, the answer is false as myAge is not equal to 50.
