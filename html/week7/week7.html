Conditional Branch Statements – checking a text input field
The first step was creating an HTML page that collects a user's name using a text input field and calls a function to display the user's name with a welcome message only when the function is called using a button. Such as "*name*, Welcome to my homepage!". As we can see the var fname = document.getElementById("fname").value; is gathering the name entered by the user and outputted to screen through the inner.HTML and <p/> element "displayDetails".
Figure 1
A JavaScript if statement was incorporated, so that if the user inputs their name, the web site will display the welcome message. The webpage was saved as practical6_T1a.html.

The next step was to modify the code above to alert the user to enter their name if the 'fname' input field have been left blank. The alert message: "Please enter your name!". If the user entered their name the welcome message would proceed to screen. The webpage was saved as Practical6_T1b.html. As we can see if if (fname == "" || x == null) the alert message will show. Conditional Branch Statements – checking radio button input
Task 2 involved the script Practical4_T1b.html to be modified so that if the user selects a university it will display the selected university information. The if statement allowed the corresponding campus values to be returned under each university. If Ulster was selected all campus values returned, if Queens was selected only the belfast campus value would be returned. If the visitor did not make the selection, an alert message will notify them to make a selection. The alert message: "Please select a university!". The webpage was saved as Practical6_T2a.html.

Conditional Branch Statements – checking grades
The next task was to create a JavaScript application that gathered four input marks of choice using the input text field to collect those values. Those values would be assigned to mark1, mark2, mark3 and mark4, then an average would be calculated using the alert function.
Figure 7
As we can see the function average(a, b, c, d) {return ((a*1 + b*1 + c*1 + d*1) /4); is used to force the input as numbers*1. Gathering them numbers through the text input id'd mark1, mark2, mark3 and mark4, then assigning those values to a,b,c,d.
The next step was to modify the code above adding some classification, using an if…else if…else style statement and replace x with the actual average mark.
a. Between 90 and 100 inclusive, output: “You got an A, your average mark is x”.
b. Between 80 and 90 inclusive, output: “You got a B, your average mark is x”.
c. Between 70 and 80 inclusive, output: “You got a C, your average mark is x.”.
d. Between 60 and 70 inclusive, output: “You got a D, your average mark is x.”.
e. Less than 60, output: “You got an E, failed, your average mark is x.”.
Figure 8
The code above was modified so that feedback could be provided along side the grade. If the grade is:
A, display: “Outstanding!”
B, display: “Very good!”
C, display: “Fairly good!”
D, display: “Doing Okay!”
E, display: “Need to work much harder!
Both an if else statement and switch statement was used within this task.
Figure 9
By providing more validation the input fields contained the attribute 'required' meaning if left blank, a message onscreen will inform the user 'Fill out this field'.
